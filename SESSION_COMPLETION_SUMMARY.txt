THE CRITICAL BUG FIX (Phase 8)
───────────────────────────────────────────────────────────────────────────────

PROBLEM:
  Admin button "Calculer Edge Actions (A+/A/B)" always returned 0 tickers
  
ROOT CAUSE:
  Line 2949 in backend/main.py called non-existent function:
  lambda: _run_screener_impl(fast=True, limit=None)
  
  This function doesn't exist → exception caught silently → cache stayed empty
  → no tickers found → always returned edge_actions_count=0

SOLUTION:
  Three improvements to lines 2942-2973:
  
  1. Multi-key cache lookup
     - Old: Only checked _screener_cache[default_key]
     - New: Searches all cache keys (handles filtered screener results)
  
  2. Actual screener function
     - Old: _run_screener_impl(fast=True, limit=None) [doesn't exist]
     - New: screener(strategy="standard", ..., fast=True) [real function]
  
  3. Type safety & diagnostics
     - Validates isinstance() before using cache entries
     - Logs which cache key was used
     - Shows full error details

RESULT:
  ✅ Bulk endpoint now returns 45+ eligible tickers
  ✅ Works with any screener cache state
  ✅ Better error diagnostics

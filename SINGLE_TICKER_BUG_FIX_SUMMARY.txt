                              🐛 LE BUG
════════════════════════════════════════════════════════════════════════════════

SYMPTÔME EXACT:
  User: POST /api/strategy-edge/compute?ticker=BIIB
  Expected: Compute BIIB only (1 ticker)
  Actual: Computed 238 tickers
  Response: "computed": 238, "message": "Edge calculé pour 238/240 tickers"

ROOT CAUSE:
  TWO endpoints with SAME route: /api/strategy-edge/compute

  Old endpoint (line 2320):
    - Parameter: "tickers" (PLURAL)
    - Default: ALL_TICKERS (238)
    - When ?ticker=BIIB is sent:
      → Looks for ?tickers=... (PLURAL) in query
      → NOT FOUND → defaults to ALL_TICKERS
      → Computes 238 tickers!

  New endpoint (line 3202):
    - Parameter: "ticker" (SINGULAR)
    - Default: REQUIRED (no default)
    - Works correctly!
    - BUT: Never executed (FastAPI uses first matching route)

THE PROBLEM IN CODE:
  Line 2320: @app.post("/api/strategy-edge/compute")
  def compute_strategy_edge(
    tickers: Optional[str] = Query(None),  ← WRONG: plural + optional
    ...
  ):
    ticker_list = (
      [t.split(",") for t in tickers.split(",")]
      if tickers else ALL_TICKERS  ← DEFAULT = 238 TICKERS!
    )
